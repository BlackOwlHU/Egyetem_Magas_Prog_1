6.
Név: IO Ibolya

Fogalom: IO (Input/Output mint mellékhatás kezelése)

Személyiség:
Ibolya a káosz királynője. Imád a külvilággal kommunikálni (billentyűzet, fájlok, konzol), de emiatt kiszámíthatatlan. Nem tiszta, mert amit csinál, az mindig hatással van a világra — viszont nélküle semmi nem történne meg igazán.

Idézet:
"Miért számolnám ki most, ha lehet, hogy soha nem is kell?"

Kódpélda:
#include <iostream>

int main() {
    int x;
    std::cin >> x;   // input (mellékhatás)
    std::cout << x;  // output (mellékhatás)
}

Ez mutatja be:
A program interakcióba lép a külvilággal.
Az eredmény nem csak bemenettől függ, nem tiszta függvény.
